Frequently Asked Questions

These are FAQ's and answers to questions you miight have about using the P&P system. reach out directly using the support button for any specific questions not covered below.

What is Predict & Pour? ›
Predict generates forecast outputs using multiple forecasting models. Pour is a Chrome extension that helps apply those forecasts inside OnePortal quickly and consistently.
Do I need to upload forecast files to the website? ›
For the current workflow, forecast files are uploaded directly inside the Pour Chrome extension. The extension stores your data locally (on your machine) and uses it when you click the POUR button in OnePortal.
How does the Pour extension know what product I’m viewing? ›
The extension reads the current product’s PDCN from the OnePortal page and finds the matching row in your uploaded file. Then it fills Week 0–12 fields automatically.
What if I click POUR and it says “No forecast found”? ›
This usually means the PDCN on the OnePortal page doesn’t exist in your uploaded forecast file (or the file hasn’t been uploaded in the extension yet). Upload the latest file in the extension popup and try again.
What OnePortal page does Pour work on? ›
Pour is designed for the OnePortal forecasting pages. If the POUR button doesn’t appear, confirm you’re on the correct OnePortal forecasting page and that the extension is enabled for that site. Refresh the page if needed.
How do I reset or clear the uploaded data? ›
Open the extension popup and click CLEAR DATA. Then upload the updated/new forecast file.
Does Predict & Pour store my data in the cloud? ›
For the extension workflow, the uploaded forecast data is stored locally (your computer) in Chrome storage on your machine. Predict and Pour cannot see your data, forecasts or files.
Need help? Click Support in the footer to email us. If you’re reporting an issue, include the PDCN you were on and a screenshot of the page you were importing into.
